Android OS Architecture

  • Linux Kernel: Handles low-level hardware interactions, security, and process management.
  • HAL: Bridges hardware and software.
  • Native Libraries: C/C++ libraries for core features (WebKit, OpenGL, etc.).
  • Android Runtime (ART): Executes app bytecode.
  • Framework: Java APIs for app development.
  • Apps: User and system applications.

Processes & Memory Management

  • Each app runs in its own process and sandbox for security.
  • Android manages memory using Linux features (OOM killer, cgroups).
  • Garbage collection in ART/Dalvik for app memory.

Security

  • App sandboxing and permissions
  • SELinux enforcement
  • Verified Boot and system integrity
